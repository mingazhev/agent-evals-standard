# Security Policy

Please do not publish an exploit that could expose held-out evaluation material,
forge trusted evidence, or bypass a non-compensable gate before maintainers can
assess it.

Report such issues through GitHub private vulnerability reporting for this
repository. Include the affected standard version, rule or schema location,
exploit sketch, impact on evaluation claims, and the smallest proposed closure.

Ordinary wording disagreements and public benchmark defects should use normal
GitHub issues.
